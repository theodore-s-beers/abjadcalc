Usage Notes

  • This follows the the common "eastern" convention. If you're working with Maghribi or otherwise nonconformist sources, then you're likely to get incorrect results. Sorry! I may try to add options at some point. (See below for more detail on the exact rules being observed.)
  • Feel free to include spaces between words, or even zero-width non-joiners; they won't affect the calculation.
  • More broadly, this program is designed to ignore any character that it doesn't recognize. If you enter text that contains one or more unrecognized characters, you'll see a brief error message, followed by the total abjad value of the parts that were counted (i.e., our best guess). It may be correct nonetheless…
  • I recommend that you not include punctuation, or diacritics, or dagger alifs, or anything else outside of the "base letters." Only the last category is counted in the abjad system, anyway, so why add complication?
  • If you're confident that you entered valid text in Arabic or Persian script, but you still received an error message, please let me know by email. It might be a bug! (There are guaranteed to be some at this stage.)

Specific Rules Followed

  • The basics of abjad numerals are covered in this Wikipedia entry, and they are observed here faithfully. If one digs a bit deeper, however, questions and ambiguities may arise…
  • The treatment of the four "Persian letters" (p, ch, zh, g) is, in fact, well established; it's explained in the same Wikipedia entry. There should be no problem using Persian text. (Urdu? Pashto? You're on your own, at least for the time being.)
  • Actually, there is one potentially confusing point in Persian: the silent hāʾ. But the answer is to treat this letter as we would any other hāʾ. (See the following note for the reasoning behind this.) Also, if you are among the proud few who use the Unicode character 06C0 (ۀ) to represent iżāfah on a silent hāʾ, that will be recognized just the same.
  • This program treats the Arabic letter tāʾ marbūṭah as if it were hāʾ and assigns it a value of 5. This is due to a general principle in abjad calculation, that we judge letters by their form, rather than by their pronunciation or other factors.
  • Similarly, alif maqṣūrah is considered equivalent to yāʾ and assigned a value of 10. (Deal with it!)
  • The treatment of hamzah is most troublesome of all, since it appears in conjunction with several different letters. But we need only continue to prioritize form. Hamzah on wāw (ؤ) is given the value of wāw (6). Hamzah on yāʾ (ئ) is given the value of yāʾ (10). Any variant of hamzah on alif, unsurprisingly, receives a value of 1. Finally, this program does assign a value of 1 to the standalone hamzah (ء), which is most often found at the end of certain Arabic words. This is a debated point; a few of the guides that I consulted were adamant that hamzah by itself should not be counted, since the appropriate value comes from the letter on which it is seated. Ultimately, I took the position of the Baha'i community. It has long been symbolically important in their faith that the name Bahāʾ (بهاء) has a numerical abjad value of 9, which requires that the final hamzah be counted. Good enough for me…
  • I should emphasize that any form of alif will be given a value of 1. This includes alif maddah, which some people have treated as a "double hamzah" with a value of 2. But here we may invoke another general principle of abjad numerals: each letter is counted exactly once. Just as shaddah is not taken into consideration, nor should alif maddah be granted double value.
